The Editors page.

Click on the “Editor login” link at the lower right corner of the page, the editor will be directed to the login page.

 Input the correct Username and Password, then click the “Enter” button, the editor will enter the “pre-edit” page.

The editor can find the wanted submission to edit by two ways: One way is to input the submission accession id in the text field and click the “Find” button, the enter the “submission edit“ page; The other option is click the “Browse All” link to enter a submission list page. By clicking the relevant accession id link, the editor will enter the submission edit page.

In the edit page, the editor can make any modification in any of the enabled update fields. After the modification, the editor can commit the update into the backend database by click the “Update” button at the foot of the page. The editor will be directed to a display page to show the detailed information of the updated submission.

In the ‘pre-edit’ page, the editor sees the original probe data submitted to the database. This original data is also entered into a ‘Probe database’ which is a subset of the Gudmap database but can be viewed independently. Details in the Probe database are cross-referenced to MGI and Genbank and updated accordingly.

It is this view of the probe database that the public sees.

By clicking on the maprobe:id in the probe details section, the editor can display the probe database.

In the submission display page, the editor can have a look at the change just made. If there's any entry the user wants to modify, clicking on the “update” button will lead the editor to the edit page again. And if the editor wants to find another submission to modify, they should click the “New Edit” button to enter the “pre-edit” page

Note: The update on the gene expression database is disabled at the present time until we fully discuss with the editors the implications of updating certain fields

However, updates are presently enabled on the Probe database.
