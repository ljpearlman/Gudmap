The Query page.

Four types of query are currently available from this page:

Search expression in components using anatomy ontology tree
Clicking on this link or its associated 'Go' button will display a page in which the user can select anatomical components from a tree structure and query the database using the selected terms. The user can select from this new page the stage range over which they wish to search by selecting from the two dropdown menus and then clicking the 'Display Tree' button.

  • Single click the tree node to select the component and double click to unselect.
  • All the selected components will be displayed in a list at the right side of the tree. When at least one component is selected, a query line will appear:
  • “Query: Find genes present/not detected in selected components’.
  • Choose ‘present’ or ‘not detected’ from the list and then click the ‘Search’ button to get the query result. 

The current Query Page can only search the ISH entries. This will be extended to Microarrays in the near future.

Search using gene symbol

At present the search is for the whole search string supplied. However testing is underway on wildcard searching on gene symbol, name and synonym. The following is the scenario for the upcoming wildcard search:

The user has the option to select gene symbol or gene name on which to search. Wildcard search is provided by selecting from the adjacent dropdown menu. The default is set to 'equals'.

A radio box provides a further filter of data. 'List genes' will return a Browse All format of entries in the database for the requested gene. 'List annotated entries' returns only annotated entries. The default sort is by component description. The column headers in the result set are:

  • Component Description
  • Component ID
  • Stage
  • Gene
  • Number of entries

Click the link(s) in the last column in the row to display the detailed information of these entries.

When stage is not specified, the result page will return a list of components expressing this gene across all stages.

Search using component ID**.

The result page will show a list of genes in the database which are expressed in this component in a specific stage. (Browse All format)

When stage is not specified, the result page will return a list of genes in the database which are expressed in this Component across all stages.

**ComponentID is a unique number assigned by the database system. Each ComponentID refers to a unique component node within the ontology trees across all stages.

A list of popular ComponentIDs can be shown by clicking the ComponentID link in the Query page.

Search using ID.

Entering the ID into the text box and clicking the corresponding 'GO' button will display that entry in the Submission Detail page format
